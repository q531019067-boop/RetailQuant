# 示例：坐标系转换文档

这是一个完整的技术参考文档示例，展示了场景驱动的写法。

参考实际文档：`Document/坐标系转换完整参考.md`

## 结构概览

```
# 文档标题

## 速查总表          ← 一张表覆盖所有转换
| 从→到 | 方法 | 公式 |

## 场景一：xxx       ← 引用块 + 流程图(TD) + 代码 + 接口位置
## 场景二：xxx
## ...

## 易错点            ← 5个坑，每个坑一段
## 常量速查          ← 底部常量表
```

## 关键设计决策

1. **速查总表 11 行** — 覆盖所有坐标系之间的转换，读者 3 秒找到答案
2. **7 个场景** — 每个场景独立成节，含 mermaid TD 流程图
3. **5 个易错点** — 从实际调试中总结的坑
4. **所有流程图用 TD** — 不用 LR（横向太挤）
5. **零行号** — 全部用文件路径 + 函数名
