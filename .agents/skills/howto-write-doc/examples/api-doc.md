# 示例：接口 API 文档

接口文档的标准写法。

## 结构

```
# 接口名称

## 快速使用
一段代码展示最常见的用法。

## 参数说明
| 参数 | 类型 | 说明 |
|------|------|------|

## 返回值
说明返回值含义。

## 接口位置
文件路径 + 函数名（不用行号）。

## 示例
完整可复制的代码片段。

## 注意事项
使用时容易踩的坑。
```

## 写法要点

1. **快速使用放最前面** — 读者复制粘贴就能跑
2. **参数用表格** — 比文字描述清晰
3. **示例要完整** — 不要省略上下文
4. **注意事项要具体** — 不要写"注意参数类型"这种废话，写"nDirection 传 -1 表示自动计算方向"
