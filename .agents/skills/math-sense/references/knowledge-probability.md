# 概率论与统计学

> 15 条核心概念。游戏中最常用的数学领域——掉落率、暴击系统、A/B测试、经济平衡都依赖它。

---

## 1. 贝叶斯定理

**公式**：`P(A|B) = P(B|A)·P(A) / P(B)`

**一句话**：用新证据更新对假设的信念——后验 ∝ 似然 × 先验。

**为什么重要**：频率学派说"概率是长期频率"，贝叶斯学派说"概率是信念程度"。贝叶斯定理给出了"看到证据B后，如何修正对A的信念"的数学公式。这是人类学习的数学模型——不断用新信息修正旧判断。

**经典例题**：一种病发病率1%，检测准确率99%。你检测阳性，真正患病的概率是多少？
- 直觉回答：99%。正确计算：P(病|阳性) = 0.99×0.01 / (0.99×0.01+0.01×0.99) ≈ 50%
- 为什么直觉错了：忽略了基础发病率极低——即使检测很准，阳性结果中也有大量假阳性。

**游戏应用**：反外挂检测——已知外挂率0.1%，检测准确率95%，被标记的玩家真正是外挂的概率只有约1.9%。这就是为什么反外挂系统需要多重证据而非单一检测。

**贝叶斯 vs 频率学派**：
- 频率学派：参数是固定的未知数，用数据估计它（MLE、置信区间）
- 贝叶斯学派：参数是随机变量，用数据更新它的分布（先验→后验）
- 游戏中：频率学派适合大样本掉落率验证；贝叶斯适合小样本A/B测试（如新副本上线后几小时内判断难度）

## 2. 大数定律

**公式**：样本均值依概率收敛到总体期望——`lim P(|X̄-μ|>ε) = 0`

**一句话**：样本越多，频率越接近概率。这是"用频率估计概率"的数学保证。

**为什么有效**：偏离期望的概率随样本量指数衰减。掷硬币10次可能有7正3反，掷10000次几乎必定接近50%。

**游戏应用**：
- 掉落率设计：单次掉落可能偏差大，但10000次掉落必然接近理论值
- 蒙特卡洛模拟的理论基础——为什么采样越多越准确
- 暴击系统：期望伤害 = 基础伤害×(1+暴击率×暴击倍率)，战斗时间越长越准确

**陷阱**："赌徒谬误"——大数定律说的是"长期频率"，不是说"短期会补偿"。连开10次铜不会增加下次出金的概率（独立事件）。

## 3. 中心极限定理（CLT）

**公式**：`(X̄-μ)/(σ/√n) → N(0,1)`（标准化样本均值趋近标准正态分布）

**一句话**：无论原始分布是什么形状，大量独立同分布变量的和/均值近似服从正态分布。这是概率论中最"魔法"的定理。

**为什么正态分布无处不在**：身高、血压、测量误差……不是因为自然界"喜欢"正态，而是因为每个观测值是大量微小独立因素的叠加，CLT告诉我们叠加的结果必然趋近正态。

**游戏设计中的应用**：
- 玩家属性分布：每个玩家的DPS是装备、技能、操作等数十个因素的叠加 → 玩家DPS分布近似正态
- 副本通关时间：受Boss技能随机、玩家暴击、网络延迟等因素叠加 → 通关时间近似正态
- 匹配系统：大量玩家的ELO分数分布近似正态，可据此设计匹配阈值

**条件**：需要独立同分布 + 有限方差。如果分布有重尾（如帕累托分布），CLT不适用。

## 4. 最大似然估计 (MLE)

**公式**：`θ̂ = argmax ∏ f(xᵢ|θ)`（选择使观测数据概率最大的参数）

**一句话**：看到数据后，最可能产生这些数据的参数值就是最佳估计。

**直观**：地上有一串脚印 → 推测最可能走出这些脚印的人的身高和步幅 → 这就是MLE。取对数把乘积变求和（对数似然），然后求导找极值。

**游戏应用**：
- 从伤害数据反推装备属性公式：记录不同攻击力下的伤害值 → 用MLE估计伤害公式的参数
- 从玩家行为数据推断匹配系统的隐藏分
- 掉落率的点估计

## 5. 假设检验与 p 值

**p值**：`p = P(数据|H₀成立)`——原假设成立时，看到当前数据或更极端数据的概率。

**最常见的误解**：p值不是"H₀成立的概率"！p=0.03 意味着"如果H₀成立，只有3%的概率看到这样的数据"——但不意味着"H₀只有3%的概率成立"。

**操作步骤**：
1. 设定原假设 H₀（如"新旧掉落率无差异"）
2. 选择显著性水平 α（通常 0.05）
3. 收集数据，计算检验统计量，得 p 值
4. p < α → 拒绝 H₀（有显著差异）；p ≥ α → 不拒绝 H₀（证据不足）

**游戏应用**：
- A/B测试：新版本掉落率是否显著不同于旧版本？
- 武器平衡：新武器DPS是否显著高于/低于基准？
- 外挂检测：某玩家的行为数据是否显著偏离正常分布？

**陷阱**：
- p > 0.05 不等于"H₀成立"，只等于"证据不足以拒绝H₀"
- 多次检验需要校正显著性水平（Bonferroni：α/m）
- 统计显著 ≠ 实际重要——需要配合效果量（Cohen's d / 相对提升百分比）

## 6. Bootstrap 自助法

**原理**：从原样本中有放回地反复抽样（每次抽n个，重复B次），用重采样分布的变异度估计统计量的抽样分布。

**为什么神奇**：不需要假设分布形态，不需要推导复杂的渐近方差——只用计算机反复重采样，就能估计任何统计量（均值、中位数、分位数、相关系数……）的标准误和置信区间。

**游戏应用**：
- 小样本A/B测试：只有100个玩家参与了新副本测试，用Bootstrap估计通关率的置信区间
- 没有正态分布假设时的稳健统计推断

## 7. 偏差-方差权衡

**公式**：`MSE = Bias² + Variance + σ²`（均方误差 = 偏差² + 方差 + 不可约噪声）

**一句话**：模型太简单 → 偏差大（系统性偏离真相）；模型太复杂 → 方差大（对训练数据敏感）。

**直观理解**：
- 高偏差 = 瞄准偏了——每次都打在靶子左边（欠拟合）
- 高方差 = 手抖——每次打的位置差很多（过拟合）
- 不可约噪声 = 靶子本身在风中晃动

**游戏应用**：
- 技能伤害公式拟合：1次多项式偏差大（太简单），10次多项式方差大（过拟合到噪声），2-3次多项式通常最佳
- 匹配系统：ELO公式太简单则偏差大（无法准确评估），太复杂则方差大（一局胜负大幅改变分数）
- 掉落率设计：过于简单的概率模型（固定概率）偏差大，过于复杂的（动态概率+保底+时间衰减）难以调试

**解决方案**：正则化（L1/L2惩罚项）、交叉验证（用验证集选模型复杂度）、集成学习（多个模型平均降低方差）

## 8. 信息熵

**公式**：`H(X) = -Σ p(x) log p(x)`（log底为2时单位是比特）

**一句话**：度量随机变量的不确定性。熵越大，确定结果需要的信息越多。

**直观**：公平硬币（p=0.5）→ H=1 bit（完全不确定，需要1 bit确定结果）；双面正面的硬币 → H=0 bit（完全确定，无需信息）。

**游戏应用**：
- 掉落系统的"惊喜度"：全均匀分布熵最大（每种掉落等可能，最不可预测）；稀有物品概率越低，系统的整体熵越小（越可预测，但稀有物品的"信息量"越大——出货时玩家更兴奋）
- 技能组合的多样性：熵越大 = 有效技能组合越多 = Meta 越多元
- 决策树AI：用信息增益（熵减）选择分裂属性
