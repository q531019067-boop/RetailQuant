## 一、证明方法

### 1. 数学归纳法

**原理**：如果一个性质对 n=1 成立，并且"对 k 成立"能推出"对 k+1 成立"，那么这个性质对所有自然数成立。这利用了自然数的**良序性**——每个非空子集有最小元，所以如果存在反例，取最小的反例会与归纳步矛盾。

**为什么有效**：它把"证明无限多个命题"转化为"证明两件事"——基础步和归纳步。这大幅降低了证明难度。

**操作步骤**：
1. 明确命题 P(n)——把"对所有n成立"改写成关于n的陈述
2. 基础步：验证 P(1)（或最小的n）为真
3. 归纳假设：假设 P(k) 为真
4. 归纳步：在假设下推导 P(k+1) 为真——这是最难的一步
5. 结论：P(n) 对所有 n 成立

**反例**：实数命题不能用归纳法。如果归纳步需要用到 P(k) 和 P(k-1)，需要强归纳法（假设 P(1)...P(k) 都成立）。

**推广**：结构归纳法——对递归定义的数据结构（树、公式）做归纳；超限归纳法——推广到超限序数。

**典型例题**：
- `1+2+...+n = n(n+1)/2`：基础步 1=1×2/2 ✓；假设 1+...+k=k(k+1)/2，则 1+...+(k+1)=k(k+1)/2+(k+1)=(k+1)(k+2)/2 ✓
- `2^n > n^2 (n≥5)`：验证 n=5 时 32>25；假设 2^k>k²，则 2^(k+1)>2k²≥(k+1)²（当 k≥3 时）

---

### 2. 反证法

**原理**：如果"结论不成立"会推出矛盾（与已知事实矛盾、或自身逻辑矛盾），那么结论必然成立。这依赖**排中律**——命题要么真要么假。

**为什么有效**：有些命题直接证明路径不清晰（如"不存在最大的素数"、"√2 不是有理数"），但假设反面却容易推导。反证法绕开了正面进攻，从敌后突破。

**操作步骤**：
1. 假设结论为假——即假设结论的否定成立
2. 从假设出发正常推导
3. 找到矛盾——矛盾可以是：与已知定理矛盾、与假设自身矛盾、推出明显荒谬的结果（如 1=0）
4. 矛盾说明假设不成立，故原结论为真

**反例**：排中律不成立的非经典逻辑中不能用（直觉主义逻辑）。如果假设的反面有多种情况，需要逐一排除。

**推广**：归谬法——反证法的一种；穷举反证——假设反面有多种情况，逐一推出矛盾。

**典型例题**：
- √2 不是有理数：假设 √2=a/b（既约分数），则 2b²=a²，a 为偶数，令 a=2c，得 2b²=4c²→b²=2c²，b 也为偶数，与"既约"矛盾
- 素数无穷多：假设有限个 p₁...pₙ，构造 N=p₁·p₂·...·pₙ+1，N 不能被任何 pᵢ 整除，故 N 是新的素数——矛盾

## 二、应用方法

### 3. 极值与优化

**原理**：光滑函数在极值点处导数为零（费马引理）。一阶导数告诉你"往哪个方向走函数值会变大/变小"，二阶导数告诉你"曲线是凹还是凸"，两者结合判定极值类型。多变量时，梯度为零是驻点条件，Hessian 矩阵的特征值符号判定极值/鞍点。

**为什么有效**：它把"找最大值"这个全局搜索问题转化为解方程 f'(x)=0——从搜索变成了代数。代价是只能找局部极值，全局极值需要在所有局部极值和边界值中比较。

**操作步骤**（单变量）：
1. 写出目标函数 f(x)
2. 求导 f'(x)，解 f'(x)=0 得临界点
3. 求二阶导 f''(x)，在每个临界点处判断：f''>0→极小值，f''<0→极大值，f''=0→不确定
4. 检查定义域边界
5. 在所有候选点中取最值

**约束优化**（多变量+约束）：拉格朗日乘数法——构造 L(x,λ)=f(x)+λ·g(x)，解 ∇L=0。

**反例**：f(x)=|x| 在 x=0 不可导但取得最小值——导数方法失效。离散优化（整数规划、组合优化）→动态规划/分支定界。非凸函数→可能陷入局部极值。

**推广**：梯度下降（数值优化）、凸优化（全局最优保证）、变分法（优化对象是函数）。在游戏中：属性分配的边际收益分析——每个属性对 DPS 的偏导数告诉你"下一步加哪个属性收益最大"。

**游戏示例**：
```
问题：800攻击力，暴击率30%，暴击伤害200%——下一步加攻击还是暴击？
→ 对 DPS 公式关于各属性求偏导
→ 比较偏导数值：哪个大就加哪个
```

### 4. 概率建模与随机验证

**原理**：自然界和游戏中的随机现象可以用概率分布描述。二项分布(Binomial)描述 n 次独立试验中成功次数；泊松分布(Poisson)描述单位时间/空间内随机事件发生次数；正态分布是大量独立微小因素叠加的极限（中心极限定理）。

**为什么有效**：概率模型把"不确定"量化为"期望值+波动范围"。你不需要知道每次掷骰子的结果，但你可以精确计算掷 1000 次的平均和方差。大数定律保证样本量足够大时频率必然收敛到概率。

**操作步骤**：
1. 识别随机源——每次试验是否独立？结果类型（二值/多值/连续）？
2. 选择分布——独立二值→二项分布；稀疏事件→泊松；大量叠加→正态
3. 计算期望值 E[X]=Σx·P(x)
4. 计算方差 Var[X]=E[(X-μ)²]，可得波动范围
5. 需要模拟验证时：蒙特卡洛→大量采样→统计频率

**反例**："伪随机"机制（保底、概率递增）不能套用独立概率模型。事件有相关性的（如暴击后下一刀必不暴击）不能用独立事件乘法。

**推广**：贝叶斯更新（先验→后验）、MCMC（任意分布采样）。游戏中：掉落率验证→CDF构建+10000次模拟→对比理论频率；暴击系统→期望伤害=基础伤害×(1+暴击率×暴击倍率)。

**游戏示例**：
```
问题：掉落率 金10%/银30%/铜60%，10000次模拟结果是否偏离理论？
→ stats_tools cdf_build → 构建理论CDF
→ stats_tools monte_carlo 10000次 → 模拟频率
→ verify 对比理论 vs 模拟 → 差异在1%内则通过
```

### 5. 傅里叶分析与周期检测

**原理**：任何周期函数可以表示为三角函数（正弦/余弦）的线性组合。这是函数空间的"坐标变换"——把函数从时域（以时间/位置为自变量）变换到频域（以频率为自变量）。FFT（快速傅里叶变换）以 O(n log n) 的时间完成这一变换。

**为什么有效**：傅里叶基 {1, cos x, sin x, cos 2x, sin 2x, ...} 构成 L² 空间的完备正交基。这意味着：(1) 任意平方可积周期函数都能展开成三角级数（完备性）；(2) 不同频率的正弦波互不干扰（正交性）——每个频率分量可以独立处理。

**操作步骤**：
1. 采集离散采样点 y₀, y₁, ..., y_{N-1}（等间隔 Δt）
2. 对数据做 FFT → 得到频谱（每个频率的振幅+相位）
3. 找主频——振幅最大的频率分量对应信号的"主导节律"
4. 可选：滤除噪声频率（低通/高通/带通滤波）
5. 从频谱反推原始信号的周期特征

**反例**：非周期信号→FFT 产生频谱泄漏；频率随时间变化的信号（非平稳）→STFT（短时傅里叶）或小波变换。采样不足（低于奈奎斯特频率）→混叠。

**推广**：短时傅里叶变换（加窗分析局部频率）、小波变换（多分辨率分析）、拉普拉斯变换（复频域，解微分方程）。游戏中：Boss 技能时间序列→FFT→发现攻击周期=12秒；DPS输出曲线→频谱→识别爆发期和疲软期的交替模式。

**游戏示例**：
```
问题：Boss技能释放时间序列 [0,12,24,36,15,27,39...] 是否有隐藏周期？
→ series_tools fft → 频谱显示主频 f=1/12 Hz
→ 结论：Boss有12秒的基础攻击周期，15秒处是特殊技能插入
```

### 6. 蒙特卡洛方法

**原理**：用随机采样逼近确定性计算。计算期望 E[f(X)] ≈ (1/N) Σ f(xᵢ)，其中 xᵢ 从分布 p(x) 中采样。收敛速度 O(1/√N) 与维度无关——在高维积分中比网格法快指数倍。

**为什么有效**：大量独立随机样本的均值必然接近真实期望（大数定律）。关键是"与维度无关"——网格法在 d 维空间中需要 M^d 个格点，而蒙特卡洛只需要 N 个样本，无论 d 多大。

**操作步骤**：
1. 定义随机模型——确定输入变量的分布
2. 生成 N 个随机样本（固定种子以保证可复现）
3. 对每个样本计算目标函数 f(xᵢ)
4. 统计：均值 = Σf/N，方差 = Σ(f-均值)²/N
5. 95% 置信区间 ≈ 均值 ± 1.96·σ/√N

**反例**：极低概率事件（如 10⁻⁶）→ 需要 10¹² 个样本才能观察到一次——用重要性采样或稀有事件模拟。有解析解的问题→蒙特卡洛是浪费。

**推广**：重要性采样（在重要区域多采样）、MCMC（马尔可夫链蒙特卡洛，对复杂分布采样）、序列蒙特卡洛（粒子滤波）。游戏中：掉落系统百万次模拟→验证理论概率；副本难度→随机生成 10000 组怪物配置→统计通关率。

**游戏示例**：
```
问题：新副本的Boss技能组合是否过于强大？
→ 定义随机模型：玩家暴击率30%、Boss技能命中率60%...
→ stats_tools monte_carlo N=50000
→ 统计玩家存活率=73%，期望通关时间=4.2分钟
→ 结论：难度适中
```
