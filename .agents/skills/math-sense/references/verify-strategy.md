# 验证策略指南

## 核心理念

大模型做数学计算不可靠。**永远不要信任单一计算结果**。用多种方法交叉验证。

## 三层验证金字塔

### 第1层：符号验证（最快）
用 `calc_sym` 做符号运算——如果 sympy 的 simplify(lhs - rhs) == 0，恒等式严格成立。
适用于：三角恒等式、代数恒等式、公式化简。

### 第2层：数值验证（通用）
用 `eval_expr` 在随机采样点上分别求值，比较误差。
适用于：任何可求值的表达式。至少取 30 个随机点。

### 第3层：统计验证（最彻底）
用 `stats_tools` 做蒙特卡洛模拟——大量随机采样后统计对比。
适用于：概率问题、随机过程、数值积分。

## 验证流程模板

### 场景 A：恒等式验证
```
1. trig_tools.py verify_identity → 符号+数值双重确认
2. eval_expr.py → 在 50 个随机点独立求值对比
3. 如果误差 > 1e-8，标记为"不成立"
```

### 场景 B：公式计算
```
1. eval_expr.py → 手工逐步代入计算
2. calc_sym.py eval → sympy 符号求值
3. 比较两者结果，允许 1e-10 浮点误差
4. 如果结果不一致，检查中间步骤
```

### 场景 C：概率分布
```
1. stats_tools.py cdf_build → 构建理论分布
2. stats_tools.py monte_carlo → 模拟 10000+ 次采样
3. 对比理论概率 vs 模拟频率
4. 偏差在 1% 内 → 公式正确
```

### 场景 D：曲线拟合
```
1. curve_tools.py fit_auto → 自动尝试多种模型
2. 比较 R²，选最优模型
3. curve_tools.py sample → 用最优模型生成预测点
4. 可视化对比原始点 vs 拟合曲线
```

## 常见陷阱

1. **浮点精度**：`0.1 + 0.2 != 0.3` 是正常的，使用 `math.isclose`
2. **除以零**：先检查分母是否接近零
3. **数值不稳定**：极大值减极大值、接近零的除数
4. **收敛判断**：迭代方法要设置最大迭代次数和容差
5. **伪随机种子**：蒙特卡洛必须固定 seed 才能复现
