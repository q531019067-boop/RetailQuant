# FFT 快速傅里叶变换：从数学推导到工程实现

> 傅里叶变换是"数学棱镜"——把信号从时域分解到频域。
> FFT（Cooley-Tukey, 1965）把这个 O(N²) 的计算降到 O(N log N)，让实时频谱分析成为可能。

## 1. 从傅里叶级数出发

任何周期为 T 的函数 f(t) 可以展开为傅里叶级数：

```
f(t) = a₀/2 + Σ[aₙ·cos(2πn t/T) + bₙ·sin(2πn t/T)]
```

其中系数由正交性给出：
```
aₙ = (2/T)∫f(t)·cos(2πn t/T)dt    (积分区间为一个周期)
bₙ = (2/T)∫f(t)·sin(2πn t/T)dt
```

**为什么正交性这么重要**：∫sin(mx)sin(nx)dx = π·δₘₙ（在[-π,π]上）。这意味着每个频率分量是"独立"的——求 aₙ 时，所有其他频率的贡献积分为零。这就像在三维空间中，(1,0,0) 和 (0,1,0) 的点积为零——频率轴彼此"垂直"。

## 2. 欧拉公式：把 sin/cos 打包成复指数

用欧拉公式 e^(iθ) = cosθ + i·sinθ，傅里叶级数可以写成更简洁的复指数形式：

```
f(t) = Σ cₙ·e^(i·2πn t/T)
cₙ = (1/T)∫f(t)·e^(-i·2πn t/T)dt
```

关键：一个复数 cₙ = |cₙ|·e^(iφₙ) 同时编码了该频率分量的**振幅**|cₙ|和**相位**φₙ。振幅告诉你"这个频率有多强"，相位告诉你"这个频率的波从什么位置开始"。

## 3. 离散傅里叶变换 (DFT)

在计算机中，我们只有 N 个离散采样点 x₀, x₁, ..., x_{N-1}（等间隔 Δt）。DFT 定义为：

```
X_k = Σ_{n=0}^{N-1} x_n · e^(-i·2πk n/N)    k = 0, 1, ..., N-1
```

直接计算这个求和——**N 个频率 k，每个需要 N 次乘加 → O(N²)。** 对于 N=1024 个采样点，需要约 100 万次运算——勉强可用。对于 N=10⁶（1秒的音频），需要 10¹² 次运算——完全不可行。

**这就是为什么需要 FFT。**

## 4. FFT 的核心思想：分治

Cooley 和 Tukey 在 1965 年发现：DFT 可以递归地一分为二。

将 N 个采样点分为偶数索引和奇数索引两组：

```
X_k = Σ x_{2n}·e^(-i·2πk(2n)/N) + Σ x_{2n+1}·e^(-i·2πk(2n+1)/N)
    = Σ x_{2n}·e^(-i·2πk n/(N/2)) + e^(-i·2πk/N)·Σ x_{2n+1}·e^(-i·2πk n/(N/2))
    = E_k + W_N^k · O_k
```

其中：
- E_k = 偶数索引的 DFT（大小为 N/2）
- O_k = 奇数索引的 DFT（大小为 N/2）
- W_N^k = e^(-i·2πk/N) —— 旋转因子（twiddle factor）

**关键递归**：一个大小为 N 的 DFT = 两个大小为 N/2 的 DFT + 乘以旋转因子后相加。继续递归——两个 N/2 各变成两个 N/4……直到大小为 1（此时 DFT 就是自身）。

## 5. 蝴蝶运算

FFT 的每一次"合并"，对应一对"蝴蝶"：

```
X_k     = E_k + W_N^k · O_k          (上半)
X_{k+N/2} = E_k - W_N^k · O_k      (下半，因为 W_N^{k+N/2} = -W_N^k)
```

这对运算称为**蝴蝶运算**（butterfly）——因为它画出来的数据流图像蝴蝶翅膀：

```
E_k ──────┬── (+) ──→ X_k
           │    ↑
           │ W_N^k
           │    ↓
O_k ──────┴── (-) ──→ X_{k+N/2}
```

整个 FFT 就是 log₂N 层蝴蝶运算的级联。每层 N/2 个蝴蝶，每个蝴蝶 1 次复数乘法和 2 次复数加法。

## 6. 复杂度分析

| 方法 | 复杂度 | N=1024 | N=10⁶ |
|------|--------|--------|-------|
| DFT（直接） | O(N²) | ~10⁶ | ~10¹² |
| FFT（Cooley-Tukey） | O(N log₂ N) | ~10⁴ | ~2×10⁷ |

对于一百万个采样点：FFT 比 DFT 快 **5 万倍**。这就是为什么实时频谱分析、图像压缩、雷达信号处理全部依赖 FFT。

## 7. 为什么必须 N 是 2 的幂？

Cooley-Tukey 算法要求 N 是 2 的幂（才能不断对半分）。如果不是 2 的幂，有两种处理方式：
- **补零**（zero-padding）：在数据末尾补零到最近的 2 的幂——最常用
- **混合基 FFT**：如果 N 有其他因子（如 3, 5），可以分解——但实现更复杂

## 8. FFT 的输出如何解读

```
输入：N 个实数采样点（时域）
输出：N/2+1 个有用的复数（频域，因为实信号的FFT对称）

X₀   = DC分量（信号的均值）
X₁   = 频率 1/(N·Δt) 的分量
X₂   = 频率 2/(N·Δt) 的分量
...
X_{N/2} = 奈奎斯特频率 = 1/(2·Δt) —— 可检测的最高频率
```

**关键**：采样频率 f_s = 1/Δt。FFT 的频率分辨率是 f_s/N。例如：采样率 1000Hz，N=1024 → 分辨率 ≈ 0.98Hz。你能区分 50Hz 和 51Hz，但区分不了 50Hz 和 50.5Hz。

## 9. 与游戏的连接

- **Boss 技能循环检测**：记录 Boss 技能释放时间序列 → FFT → 频谱显示峰值在 f=1/12 Hz → Boss 有 12 秒的基础攻击周期

- **DPS 节奏分析**：记录每秒伤害输出 → FFT → 识别爆发期（高频能量大）和疲软期（低频为主）

> FFT 的故事：傅里叶在 1807 年提出"任何函数都能分解为正弦波"时被拉格朗日嘲笑"不严格"。160 年后，Cooley 和 Tukey 用分治算法让这个想法在计算机上实时运行。今天你听的每首 MP3、看的每张 JPEG、用的每次 WiFi——背后都是 FFT。
