# 误差控制与精度管理

## 浮点误差的本质

IEEE 754 双精度（Python float）只有约 15-17 位有效数字。每次运算可能引入 1e-16 量级的相对误差。

```
0.1 + 0.2 = 0.30000000000000004  # 不是 0.3！
```

## 比较浮点数的正确方式

```python
import math

# 绝对容差（适合比较接近零的数）
abs(a - b) < 1e-10

# 相对容差（适合比较一般数值）
math.isclose(a, b, rel_tol=1e-9, abs_tol=1e-12)
```

## 数值方法中的误差

| 方法 | 误差来源 | 控制方式 |
|------|---------|---------|
| 有限差分导数 | 截断误差 O(h²) + 舍入误差 O(ε/h) | h ≈ 1e-5~1e-8 |
| 数值积分（梯形法） | 截断误差 O(h²) | 增加分段数 |
| 牛顿迭代 | 局部收敛（初值敏感） | 配合二分法保证全局收敛 |
| FFT | 频谱泄漏 | 加窗函数 |
| 蒙特卡洛 | 统计误差 O(1/√n) | 增加样本数或重要性采样 |

## 高精度需求场景

当默认精度不够时：
- `eval_expr.py --precision 50` → 使用 Decimal（50位有效数字）
- `calc_sym.py` 默认使用 sympy 的符号计算（无限精度）
- 对于关键结果，用 sympy 做有理数运算避免浮点

## 误差传播

如果 z = f(x, y)，且 x、y 各有误差 Δx、Δy：
```
Δz ≈ |∂f/∂x|·Δx + |∂f/∂y|·Δy  （一阶近似）
```

用 `calc_sym.py gradient` 计算偏导数来估计传播的误差。
