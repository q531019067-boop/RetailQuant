# LaTeX 数学公式修复 — 前后对比示例

## 1. 千分位 `{,}` 移除

### 修复前
```markdown
服务器最大承载人数为 $1{,}000{,}000$ 人。
预计收入 $¥50{,}000$ 元。
$$S = 1{,}024 \times 1024$$
```

### 修复后
```markdown
服务器最大承载人数为 $1000000$ 人。
预计收入 $¥50000$ 元。
$$S = 1024 \times 1024$$
```

### 说明
- `{,}` 在某些 LaTeX 渲染器中无法解析，会导致公式显示失败
- 直接移除逗号即可，或改用科学计数法（如 `$10^6$`）

---

## 2. 表格中的 `\mid` 替换为 `\vert`

### 修复前
```markdown
| 变量 | 取值范围 | 说明 |
|------|----------|------|
| x | $\{x \mid x > 0\}$ | 正实数 |
| y | $\{y \mid y \in \mathbb{Z}\}$ | 整数 |
```

### 修复后
```markdown
| 变量 | 取值范围 | 说明 |
|------|----------|------|
| x | $\{x \vert x > 0\}$ | 正实数 |
| y | $\{y \vert y \in \mathbb{Z}\}$ | 整数 |
```

### 说明
- `\mid` 在表格中可能被 Markdown 解析器误认为列分隔符，导致表格错乱
- 使用 `\vert` 替代，它不会与表格语法冲突

---

## 3. `\Sigma` 替换为 `\sum`

### 修复前
```markdown
总伤害公式：
$$\text{总伤害} = \Sigma_{i=1}^{n} d_i$$

每轮伤害为 $d_i$，共 $n$ 轮，$\Sigma$ 为求和符号。
```

### 修复后
```markdown
总伤害公式：
$$\text{总伤害} = \sum_{i=1}^{n} d_i$$

每轮伤害为 $d_i$，共 $n$ 轮。
```

### 说明
- `\Sigma` 是大写希腊字母 Σ，上下限显示在右侧（行内）或上下方（块级），但视觉上不如求和符号
- `\sum` 是专用的求和运算符 ∑，上下限位置更合理
- 如果确实需要大写希腊字母 Σ（而非求和），保留 `\Sigma`

---

## 4. 复杂行内公式转为块级或文字

### 修复前
```markdown
概率 $P(\text{从 $n$ 个中选 $k$ 个}) = \frac{C_n^k}{2^n}$ 表示组合数的比例。
计算 $\frac{\text{成功次数}}{\text{总次数}} = \hat{p}$ 得到估计值。
```

### 修复后
```markdown
从 $n$ 个中选 $k$ 个的概率为：

$$
P = \frac{C_n^k}{2^n}
$$

计算估计值 $\hat{p} = \frac{\text{成功次数}}{\text{总次数}}$。

或者：估计值 $\hat{p}$ 等于成功次数除以总次数。
```

### 说明
- 行内使用 `\frac` 会导致行高膨胀，与周围文字不协调
- 行内公式包含中文 `\text{}` 会进一步加重渲染问题
- 复杂公式改用块级 `$$...$$`，或将文字部分移到公式外

---

## 5. `\mathrm{中文}` 替换为 `\text{中文}`

### 修复前
```markdown
定义 $\mathrm{临界值}$ 为：
$$f(x) = \mathrm{当} x > 0 \mathrm{时成立}$$

设 $\textrm{最大值} = M$，$\textrm{最小值} = m$。
```

### 修复后
```markdown
定义 $\text{临界值}$ 为：
$$f(x) = \text{当 } x > 0 \text{ 时成立}$$

设 $\text{最大值} = M$，$\text{最小值} = m$。
```

### 说明
- `\mathrm{}` 和 `\textrm{}` 使用 Roman 字体，不包含中文字符集
- 中文会显示为方框或乱码
- 替换为 `\text{}` 即可解决

---

## 6. `$$` 未独占行

### 修复前
```markdown
总和为 $$S = \sum_{i=1}^{n} a_i$$ 其中 $n$ 为项数。
下面展示完整推导过程：
$$\frac{d}{dx}[f(g(x))] = f'(g(x)) \cdot g'(x)$$
这个公式很重要。
```

### 修复后
```markdown
总和为：

$$
S = \sum_{i=1}^{n} a_i
$$

其中 $n$ 为项数。

下面展示完整推导过程：

$$
\frac{d}{dx}[f(g(x))] = f'(g(x)) \cdot g'(x)
$$

这个公式很重要。
```

### 说明
- 块级公式 `$$...$$` 必须独占一行
- 前后各留一个空行，确保渲染器正确识别公式边界
- 公式前后的文字移出公式范围

---

## 7. 公式中的空格问题

### 修复前
```markdown
设 $ x = 10 $，$ y=20 $。
当 $ a + b = c $ 时成立。
```

### 修复后
```markdown
设 $x = 10$，$y = 20$。
当 $a + b = c$ 时成立。
```

### 说明
- `$` 和公式内容之间不应有空格
- 正确格式：`$formula$`
- 公式内部的空格（如 `$a + b$` 中 `+` 两侧）是正常的，由 LaTeX 自动处理
