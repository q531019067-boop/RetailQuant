# LaTeX 数学公式故障排除指南

## 问题 1：公式显示为原始 `$...$` 文本

### 症状
公式没有被渲染，页面上直接显示 `$E = mc^2$` 这样的原始文本。

### 原因
Markdown 渲染器未启用数学公式支持。

### 解决方案

#### Typora
1. 打开 **偏好设置**（Ctrl + ,）
2. 进入 **Markdown** 选项卡
3. 勾选 **"内联公式"**（Inline Math）
4. 重启 Typora

#### VS Code
1. 安装 **Markdown+Math** 扩展（`goessner.mdmath`）
2. 或者安装 **Markdown Preview Enhanced**（`shd101wyy.markdown-preview-enhanced`）
3. 在 VS Code 设置中添加：
   ```json
   "mdmath.delimiters": "dollars"
   ```

#### GitHub
- GitHub 原生支持 LaTeX 数学公式（使用 MathJax）
- 如果不渲染，检查是否在 `.md` 文件中（README.md、issues、discussions 等）
- 注意：GitHub 的公式支持有限制，某些高级命令可能不支持

#### Hugo / Hexo 等静态网站生成器
- 需要在配置中启用 MathJax 或 KaTeX 支持
- Hugo：在模板中引入 MathJax/KaTeX 脚本
- Hexo：安装 `hexo-renderer-markdown-it` + `markdown-it-katex` 插件

## 问题 2：部分公式能渲染，部分不能

### 症状
同一个文档中，一些公式正常显示，另一些显示为原始文本或报错。

### 排查清单

#### 检查 `{,}` 千分位模式
```markdown
<!-- 这会导致渲染失败 -->
$1{,}000{,}000$

<!-- 修复 -->
$1000000$
```

#### 检查未匹配的 `$`
```markdown
<!-- 奇数个 $ 会导致后续公式全部失效 -->
价格是 $100 元
后续公式 $x = y$ 也会受影响

<!-- 修复 -->
价格是 \$100 元
后续公式 $x = y$ 正常显示
```

#### 检查表格中的 `|` 或 `\mid`
```markdown
<!-- 表格中的 | 会被解析为列分隔符 -->
| 条件 | $x \mid x > 0$ |

<!-- 修复 -->
| 条件 | $x \vert x > 0$ |
```

#### 检查 `$$` 是否独占行
```markdown
<!-- $$ 没有独占行会导致解析错误 -->
公式为 $$x = y$$ 其中 x 是变量。

<!-- 修复 -->
公式为：

$$
x = y
$$

其中 x 是变量。
```

## 问题 3：公式破坏了表格布局

### 症状
- 表格列数突然增多
- 表格行显示错乱
- 公式内容被分割到不同单元格

### 原因
公式中的 `|` 或 `\mid` 被 Markdown 解析器当作了表格列分隔符。

### 解决方案
```markdown
<!-- 错误写法 -->
| 变量 | 取值范围 |
|------|----------|
| x | $x \mid x > 0$ |

<!-- 正确写法：用 \vert 替代 -->
| 变量 | 取值范围 |
|------|----------|
| x | $x \vert x > 0$ |

<!-- 更好的做法：简化公式，避免在表格中用竖线 -->
| 变量 | 取值范围 |
|------|----------|
| x | 正实数 $x > 0$ |
```

## 问题 4：中文字符在公式中显示乱码

### 症状
公式中的中文变成了方框、问号或其他乱码字符。

### 原因
使用了 `\mathrm{}` 或 `\textrm{}` 包裹中文，这两个命令只支持 ASCII 字符。

### 解决方案
```markdown
<!-- 错误 -->
$\mathrm{最大值}$
$\textrm{最小值}$

<!-- 正确 -->
$\text{最大值}$
$\text{最小值}$

<!-- 最佳实践：把中文放到公式外 -->
最大值为 $M$，最小值为 $m$。
```

## 问题 5：行内公式太宽，撑破了行高

### 症状
- 包含公式的那一行行距明显变大
- 行内公式与周围文字高度不一致
- 分数 `\frac` 在行内显示被压缩

### 原因
复杂公式（特别是包含 `\frac`、`\sum` 等有上下结构的命令）在行内模式下会导致行高膨胀。

### 解决方案
```markdown
<!-- 不推荐：行内复杂公式 -->
计算 $\frac{a + b}{c + d}$ 的值。

<!-- 推荐：改为块级公式 -->
计算以下表达式的值：

$$
\frac{a + b}{c + d}
$$

<!-- 或者简化行内表达式 -->
计算 $(a + b) / (c + d)$ 的值。
```

### 决策参考
- 行内公式适合：变量名 `$x$`、简短表达式 `$a + b$`、上标下标 `$x^2$`
- 应改块级公式：分数 `\frac`、求和 `\sum`、积分 `\int`、任何导致行高变化的公式

## 问题 6：`\Sigma` 和 `\sum` 混淆

### 症状
求和符号的上下限位置不对，显示为右侧而不是上下方。

### 原因
- `\Sigma` 是大写希腊字母 Σ，上下限显示在右侧
- `\sum` 是求和运算符 ∑，上下限显示在上下方

### 解决方案
```markdown
<!-- 错误 -->
$\Sigma_{i=1}^{n} a_i$    <!-- 上下限在右边 -->

<!-- 正确 -->
$\sum_{i=1}^{n} a_i$      <!-- 上下限在上下方 -->
```

## 平台特定设置

### Typora 完整配置
1. 偏好设置 → Markdown → 勾选「内联公式」和「块级公式」
2. 偏好设置 → 通用 → 主题选择支持数学渲染的主题
3. 如果仍有问题，检查 Typora 版本（建议 >= 1.0）

### VS Code 配置
推荐扩展组合：
- **Markdown All in One**（基础 Markdown 支持）
- **Markdown+Math**（数学公式渲染）
- **Markdown Preview Enhanced**（增强预览）

settings.json 配置：
```json
{
    "mdmath.delimiters": "dollars",
    "mdmath.macros": {},
    "markdown-preview-enhanced.mathRenderingOption": "MathJax"
}
```

### Obsidian
- 默认支持 MathJax，无需额外配置
- 设置 → 编辑器 → 确认「使用 LaTeX 语法」已开启

### Notion
- 使用 `/math` 块或 `$$...$$` 语法
- 注意：Notion 的 LaTeX 支持有限，某些命令不支持
