# LaTeX 数学公式中文兼容性指南

## 概述

在 Markdown 的 LaTeX 数学公式中使用中文需要特别注意。不同的命令对中文的支持程度不同，不同渲染器（MathJax、KaTeX）的表现也有差异。

## 各命令的中文支持情况

### `\text{}` — 推荐

- **MathJax**: 支持中文 ✓
- **KaTeX**: 支持中文 ✓（需确保字体配置正确）
- **适用场景**: 在公式中插入少量中文说明

```markdown
$P(\text{成功}) = 0.5$
```

**注意事项**:
- 字体对齐可能与纯英文不同
- 过长的中文文本会导致公式行宽异常
- 某些浏览器中可能需要额外字体配置

### `\mathrm{}` — 不支持中文

- **MathJax**: 中文显示为乱码或方框 ✗
- **KaTeX**: 中文显示为方框 ✗
- **原因**: `\mathrm` 使用 Roman 字体族，该字体不包含中文字符集

```markdown
<!-- 错误 -->
$\mathrm{最大值}$

<!-- 显示效果 -->
□□□    <!-- 乱码 -->
```

### `\textrm{}` — 不支持中文

- **表现**: 与 `\mathrm{}` 相同，不支持中文 ✗
- **原因**: `\textrm` 也是指定 Roman 字体

```markdown
<!-- 错误 -->
$\textrm{最小值}$

<!-- 显示效果 -->
□□□    <!-- 乱码 -->
```

### `\mbox{}` — 有限支持

- **MathJax**: 部分版本支持中文
- **KaTeX**: 不支持 ✗
- **不推荐**: 兼容性差，应使用 `\text{}` 替代

## 最佳实践

### 原则 1：中文尽量放在公式外

这是最可靠的方案，避免一切兼容性问题。

```markdown
<!-- 推荐 -->
设 $x$ 为最大值，$y$ 为最小值。
当 $x > y$ 时，条件成立。

<!-- 避免 -->
设 $x \text{为最大值}$，$y \text{为最小值}$。
```

### 原则 2：必须在公式内时使用 `\text{}`

如果中文必须出现在公式内部，只使用 `\text{}` 包裹。

```markdown
<!-- 可接受 -->
$P(\text{事件发生}) = \frac{n}{N}$

$C = \begin{cases} 1 & \text{满足条件} \\ 0 & \text{不满足条件} \end{cases}$
```

### 原则 3：保持行内公式简短

包含中文 `\text{}` 的行内公式不应过长，避免与 `\frac` 等命令组合使用。

```markdown
<!-- 不推荐：行内公式太复杂 -->
概率 $P(\text{从 $n$ 个中选 $k$ 个}) = \frac{C_n^k}{2^n}$ 为...

<!-- 推荐：改为块级公式或分离文字 -->
从 $n$ 个中选 $k$ 个的概率为：

$$
P = \frac{C_n^k}{2^n}
$$
```

### 原则 4：中英文混合时注意间距

公式内的中英文混合可能导致间距不自然。

```markdown
<!-- 可能间距不佳 -->
$x \text{的} \text{取值}$

<!-- 更好的做法 -->
$x$ 的取值
```

## 好的模式 vs 坏的模式

### 好的模式

```markdown
<!-- 1. 中文在公式外 -->
设 $a = 1$，$b = 2$，则 $a + b = 3$。

<!-- 2. 用 \text{} 简短插入 -->
$f(x) = \begin{cases} x & \text{当 } x \geq 0 \\ -x & \text{当 } x < 0 \end{cases}$

<!-- 3. 用变量代替中文 -->
令 $M$ 表示最大值，$m$ 表示最小值。$M - m$ 为极差。

<!-- 4. 块级公式中简短的 \text{} -->
$$
\text{最优解} = \arg\min_{x} f(x)
$$
```

### 坏的模式

```markdown
<!-- 1. 使用 \mathrm{} 包含中文（乱码） -->
$\mathrm{概率密度函数}$

<!-- 2. 使用 \textrm{} 包含中文（乱码） -->
$\textrm{期望值}$

<!-- 3. 行内公式中过长的中文 -->
$x$ 的取值范围是 $\{x \mid x \text{是正整数且} x < 100\}$

<!-- 4. 行内复杂公式含中文 + \frac -->
结果为 $\frac{\text{成功的次数}}{\text{总试验次数}} = p$
```

## 不同渲染器的差异

### MathJax (v2/v3)
- `\text{中文}` 正常工作
- 字体回退机制较好
- 需要正确配置字体路径（某些自部署场景）

### KaTeX
- `\text{中文}` 基本正常
- 字体渲染可能与 MathJax 有细微差异
- 性能更好，但功能子集较小

### GitHub
- 使用 MathJax 渲染
- `\text{中文}` 可用
- 复杂公式支持有限

### Typora
- 可选 MathJax 或 KaTeX（取决于版本和配置）
- 本地渲染，字体依赖系统字体
- Windows 上中文显示通常正常

## 自动化检查

使用 `check_math.py` 脚本可以自动检测以下中文兼容性问题：
- `\mathrm{}` 包含中文（WARNING）
- `\textrm{}` 包含中文（WARNING）
- 行内公式中 `\frac` + `\text{}` 含中文（WARNING）

```bash
python scripts/check_math.py document.md
```

使用 `fix_math.py` 可以自动修复：
- `\mathrm{中文}` → `\text{中文}`
- `\textrm{中文}` → `\text{中文}`

```bash
python scripts/fix_math.py document.md --in-place
```
