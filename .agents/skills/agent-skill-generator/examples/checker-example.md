# 检查器类型Skill示例

## 功能说明
本示例展示如何生成一个检查器类型的Skill，用于检查配置文件的正确性和完整性。

## 输入
```
检查配置文件是否正确，比较相互之间的关系，找出有问题的部分
```

## 输出
```
正在生成skill: config-checker (类型: checker)
创建目录结构...
编写SKILL.md文件...
生成脚本文件...
Skill生成完成！
生成路径: K:\Sword3-products\trunk\tools\SKILLS\config-checker
```

## 生成的文件结构
```
config-checker/
├── SKILL.md              # 检查器使用指南
├── scripts/              # [必需] 检查脚本
│   └── check.py          # 检查执行脚本
├── examples/             # [可选] 示例目录
└── references/           # [可选] 参考资料
```

## 使用场景

### 场景1：ID唯一性检查
**用户**：检查配置中是否有重复的ID

**检查器**：
1. 扫描所有配置文件
2. 检查ID唯一性
3. 报告重复项

### 场景2：引用完整性检查
**用户**：检查引用是否正确

**检查器**：
1. 提取所有引用
2. 验证引用目标存在
3. 报告无效引用

## 检查项目
- [ ] 文件格式正确
- [ ] 必填字段完整
- [ ] 数据类型正确
- [ ] 取值范围合理
- [ ] 引用的ID存在
- [ ] 依赖关系正确
- [ ] 无循环依赖

## 核心特点
- 全面的配置检查
- 关联性验证
- 自动生成检查报告
- 提供修复建议
