# 策略回测链路测试文档

> 测试日期：2026-06-20  
> 目标：验证“板块选池 → 数据准备 → **多策略评分** → 组合模拟炒股 → 策略收益矩阵”整条链路可运行，并确认策略计算严格使用 as-of 日期之前的数据。

本文档对应 `docs/大纲.md` 中的**研究回测轨**，描述从选池到模拟的端到端流程。

**链路阶段划分：**

| 步骤 | 脚本 | 做什么 |
|------|------|--------|
| 1 选池 | `select_board_pool.py` | 板块与候选股 |
| 1.5 拉数 | `fetch_hist_for_pool.py` | 历史 K 线落盘 |
| **2 多策略评分** | `score_stock.py` / `workflow.score_stock_strategies` | 对每只股票运行多个注册策略，产出 score 与信号 |
| 3 组合模拟 | `simulate_trading.py` | 逐日消费评分结果，模拟持仓与成交 |

`MultiFactor` 属于**步骤 2（多策略评分阶段）**，与 `MovingAverageCross`、`GridMartingale` 等并列；其因子设计见 `docs/多因子选股回测系统.md`。组合模拟（步骤 3）只读取各策略在评分阶段的输出，不再单独定义策略逻辑。

## 0. 与项目定位的对齐检查

| 项目定位要求 | 本链路覆盖情况 | 说明 |
|-------------|---------------|------|
| A 股个人量化、本地优先 | ✅ | 默认读 Parquet/SQLite，不隐式联网 |
| 策略引擎是核心 | ✅ 部分 | 复用 `rquant/strategy/` 注册策略；默认仅 4 个，非全部 10 个 |
| 板块/标的池选股 | ✅ | `select_board_pool.py` 按流动性/热度/波动选池 |
| 历史验证（回测/模拟） | ⚠️ 分阶段 | 烟测 2 天验证链路；**完整测试窗口为 1 年**（见 §3.1） |
| 防未来函数 | ✅ | as-of K 线切片 + 数据拉取与策略判断分离 |
| 实盘看板（持仓/自选/Web） | ❌ 不在范围 | 由 `python app.py` + `rquant/web/` 覆盖，本链路不测 |

**策略覆盖缺口（有意为之，非 bug）：** 默认策略为 `MovingAverageCross`、`RsiMeanReversion`、`GridMartingale`、`MultiFactor`，**不含** `VpBreakout`、`DonchianTurtle`、`ScenarioRouter` 等其余注册策略。需要时可传 `--strategies VpBreakout ...` 扩展。

**完整测试前置条件：** 120 只候选池需先批量拉数（`--to` 至少到模拟结束日 `2026-06-18`），再跑 1 年模拟；仅 2 天烟测不能代表策略真实收益。

## 1. 测试目标

测试分两层：

1. **烟测（已执行）**：验证脚本可运行、输出结构化结果，窗口 2 天。
2. **完整测试（目标规模）**：20 板块 × 6 股 ≈ 120 只候选池，模拟窗口 **2025-06-18 ~ 2026-06-18**（1 年）。

烟测验证链路完整性：

1. 大盘/板块选股脚本可以按指定日期筛选高流动性、热门、高波动的板块与候选股。
2. 数据准备脚本可以复用仓库已有 `scripts/fetch_hist.py` 拉取历史日频数据，并落盘到本地 Parquet、SQLite K 线缓存和 pool 表。
3. 多策略评分脚本可以对指定股票、指定日期、指定 lookback 窗口并行运行多个注册策略（含 `MultiFactor`）。
4. 组合模拟脚本在时间范围内逐日消费各策略的评分结果，按本金和持仓约束模拟交易，并输出策略收益矩阵。
5. 所有策略判断只使用 `date <= as_of` 的数据，允许数据拉取到测试结束日、当天或更远日期，但策略不可看到传入日期之后的未来数据。

## 2. 测试对象

### 2.1 核心模块

- `rquant/research/workflow.py`
  - 读取 `.tab` 板块/股票关系表。
  - 读取本地 Parquet/业务 K 线。
  - 执行 as-of 日期切片。
  - 计算板块和个股流动性、波动、热度分数。
  - 对单股运行**多策略评分**（`score_stock_strategies`）。
  - 对候选池运行组合模拟（消费评分结果）。

### 2.2 命令行脚本

- `scripts/select_board_pool.py`
  - 步骤 1：板块/股票池选择。
- `scripts/fetch_hist_for_pool.py`
  - 步骤 1.5：按股票池拉取历史行情并落盘。
- `scripts/score_stock.py`
  - 步骤 2：**多策略评分**（单股、单日、多策略并行打分）。
- `scripts/simulate_trading.py`
  - 步骤 3：组合模拟（逐日调用评分结果，非策略定义阶段）。

### 2.3 多策略评分阶段的参与策略

本阶段由 `score_stock_strategies()` 对同一只股票、同一 as-of 日并行调用多个策略。默认列表（`DEFAULT_STRATEGIES`）：

- `MovingAverageCross`
- `RsiMeanReversion`
- `GridMartingale`
- `MultiFactor` — 使用 `MultiFactor.score()` 输出连续分数；详见 `docs/多因子选股回测系统.md`

未纳入默认、但已通过 `@register` 注册、可用 `--strategies` 手动加入的策略示例：

- `VpBreakout`（量价突破，见 `rquant/strategy/volume_breakout/vp_breakout.py`）
- `DonchianTurtle`、`ScenarioRouter`、`ChanLun2B` 等（完整清单见 `docs/大纲.md` 策略层）

## 3. 测试条件

### 3.1 交易条件

- 本金：`100000`
- **完整测试窗口：`2025-06-18` 到 `2026-06-18`（1 年）**
- 烟测窗口：`2025-06-18` 到 `2025-06-19`（2 天，仅验证链路）
- 单股评分 lookback：`365` 自然日
- 股票池选择日期：`2025-06-18`
- 默认策略模拟最大持仓数：`5`
- 手续费、印花税、滑点沿用回测配置：
  - 佣金：双向万 2.5，最低 5 元
  - 印花税：卖出单向 0.05%
  - 滑点：默认 1 bp

### 3.2 数据条件

选池和模拟默认不隐式联网，优先使用本地 Parquet 缓存。

完整测试前，对 120 只候选池补全历史数据（`--from` 需覆盖 lookback，`--to` 至少到模拟结束日）：

```powershell
python scripts/fetch_hist_for_pool.py --pool-file results/research/board_pool_2025-06-18.json --from 2024-06-18 --to 2026-06-18
```

也可以拉取到当天或测试结束日之后（策略仍只用到 `as_of` 及之前的数据）：

```powershell
python scripts/fetch_hist_for_pool.py --pool-file results/research/board_pool_2025-06-18.json --from 2024-06-18 --to 2026-06-20
```

注意：即使数据拉取到更晚日期，策略评分仍按 `as_of` 做 `date <= as_of` 切片。

## 4. 防未来函数设计

### 4.1 数据准备和策略判断分离

数据准备阶段可以拉到测试结束日、当天或更远日期，用于补齐本地缓存。

策略判断阶段必须经过 as-of 切片：

```python
sliced = df[(df["date"] >= start_ts) & (df["date"] <= asof_ts)]
```

单股评分和组合模拟均使用该切片逻辑。

### 4.2 默认不隐式联网

为保证烟测稳定和复现，研究链路默认不在评分/模拟过程中自动联网拉数。

需要联网时必须显式指定：

- 单股评分：`--fetch-missing`
- 组合模拟：`--fetch-missing`
- 或单独运行：`scripts/fetch_hist_for_pool.py`

这样可以避免“评分过程中临时拉到更晚数据但未明确记录”的不透明行为。

## 6. 测试结果解释

### 6.1 多策略评分有信号、组合模拟无交易的原因

多策略评分（步骤 2）只回答“某只股票在某个 as-of 日、某个策略下是否有信号”。

组合模拟脚本还会受到以下约束：

- 候选池来源不同。
- 买入需要在组合横截面中被选中。
- 两日窗口极短，只有一个可执行的 T+1 交易机会。
- 模拟阶段默认只使用本地缓存，不会为了凑足样本临时联网。

因此，多策略评分阶段出现买入信号，而组合模拟（步骤 3）最终没有成交，是允许的。

### 6.2 烟测与完整测试的差异

烟测目标是链路验证，**不能**代替 1 年 × 120 只的完整测试。

完整测试前应先批量拉数，再跑 1 年模拟：

```powershell
python scripts/fetch_hist_for_pool.py --pool-file results/research/board_pool_2025-06-18.json --from 2024-06-18 --to 2026-06-18
python scripts/simulate_trading.py --start 2025-06-18 --end 2026-06-18 --capital 100000 --pool-file results/research/board_pool_2025-06-18.json --out-dir results/research/simulation_2025-06-18_2026-06-18
```

可选：加入 `VpBreakout` 等与 Web 看板一致的突破类策略：

```powershell
python scripts/simulate_trading.py --start 2025-06-18 --end 2026-06-18 --capital 100000 --pool-file results/research/board_pool_2025-06-18.json --strategies MovingAverageCross RsiMeanReversion GridMartingale MultiFactor VpBreakout --out-dir results/research/simulation_2025-06-18_2026-06-18
```

## 8. 当前结论

策略回测链路已经具备可复现的模块化执行能力，**烟测已通过**：

- 第一步可以生成板块/股票候选池。
- 第二步可以在**多策略评分阶段**对指定股票、指定日期并行运行多个策略。
- 第三步可以在给定时间范围内模拟策略组合收益。
- 数据拉取和策略判断已解耦。
- 防未来函数约束已落实为 as-of K 线切片。

**尚未满足项目定位的完整验证**（待执行）：

- 1 年窗口（2025-06-18 ~ 2026-06-18）× 120 只候选池的完整模拟。
- 默认 4 策略与 Web 看板 10 策略之间的覆盖对齐（尤其 `VpBreakout` 等突破类策略）。
- 完整测试产出有意义的 `strategy_matrix.csv`（烟测 2 天 0 成交属预期，不代表策略无效）。

下一步：先 `--to 2026-06-18` 批量拉数，再跑 §5.4 / §7.4 中的 1 年模拟命令；如需与看板策略一致，用 `--strategies` 扩展策略列表。
