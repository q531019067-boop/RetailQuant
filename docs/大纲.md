## 各层速查

### ★ `strategy.py` — 核心（~110 行）

项目和其他看股票软件的唯一区别所在。**改动永远先从这里下手。**

| 函数 | 干什么 |
|------|--------|
| `chanlun2b_signal(code, name, sector, df)` | 缠论 2B 买入：站上 MA5 + MA5>MA20 + 昨日确认上穿 → `Signal` |
| `buyhold_signal(code, name, sector, df)` | 低吸：现价 < MA60×0.95 → `Signal` |
| `scan_stock(code, name, sector, df)` | 对单只股票跑上面两个策略，收集命中 |
| `sell_signal(position, df)` | 卖出：止损-7% / 止盈+15% / 跌破 MA60 |

**Signal 字段：** `code, name, sector, strategy, current_price, suggested_buy, stop_loss, take_profit, reason, confidence`

**改策略 → 改这里面。加策略 → 写新函数 + 在 `scan_stock()` 加一行调用。阈值全硬编码在这。**

依赖：仅 `pandas`，不碰 HTTP、不碰文件。

### 数据与业务支撑层

| 文件 | 行数 | 干什么 | strategy 用它做什么 |
|------|------|--------|---------------------|
| `data.py` | ~95 | K 线 wrapper、自选股 CRUD、标的池、内存股票字典 | 提供 `fetch_kline()` 拿 OHLC |
| `portfolio.py` | ~110 | 持仓/交易/快照的 JSON 存储 | 信号触发的买卖结果记到这里 |
| `board.py` | ~142 | 30 只 ETF 映射、行情排行、2 分钟缓存 | 提供市场宽度（哪热哪冷） |
| `datasources.py` | 278 | Sina HTTP → JSON 缓存、健康度、failover | data.py/board.py 的底层数据源 |

**这些模块是被动适配层 — strategy 不需要知道它们内部怎么变。** 加新数据源只在 `datasources.py` 写新类 + 注册到 `pool`，data.py/board.py 不用动。

### 入口与展示层

| 文件 | 干什么 | 改它的时候 |
|------|--------|-----------|
| `app.py` | Flask 11 路由 + 薄表单校验 + 启动 | 加新按钮/新 API 才动 |
| `templates/index.html` | 看板 HTML | 前端展示需求变才动 |
| `static/style.css` | 样式 | 调 UI 才动 |

**app.py 不拥有任何业务逻辑。** 路由函数做三件事：解析参数 → 调 strategy/data/portfolio → 塞进模板。

## 股票代码格式

小写 `sh`/`sz` + 6 位数字。例：`sh600519`、`sz000001`。

K 线 DataFrame 列名：`date, open, high, low, close, volume`（全小写）。

## 陷阱

1. **Sina 接口会挂** — failover 自动标记 unhealthy 60 秒，下游要容错空返回
2. **双层缓存** — `board.py` 2 分钟 + `SinaQuoteSource` 30 秒，改缓存两个一起看
3. **单实例，无锁** — JSON 并发写可能丢数据，但单用户本地用不构成实际风险
4. **`_test_*.py` 不是正式模块** — 开发探针，忽略
5. **前端 squarify 已删** — Treemap 坐标纯后端 Python 计算
