```text

║     10 个策略 + 注册中心 + 兼容 scan API            ║
╠════════════════════════════════════════════════════╣
║  rquant/business/  业务层                           ║

║  app.py  strategy.py  根目录兼容入口                 ║
╚════════════════════════════════════════════════════╝
```

### `rquant/log/` — 统一日志

基于 loguru 的三输出日志模块：stderr（控制台/waitress）+ 落盘文件（自动轮转）+ 内存 ring buffer（Web 展示）。`init_logging()` 幂等，其他模块通过 `info/warning/error/debug` 函数打日志。

### `config/` — 全局配置

`from config import config` 获取单例，从 `config.toml` 读取（tomllib，Python 3.11+），加载时校验必需项，缺失报错退出。支持环境变量覆盖（如 `RQUANT_PORT`）。

## 各层速查

### ★ `rquant/strategy/` — 策略核心

项目和普通看盘软件的主要区别在这里。新增策略时改策略包，不把策略逻辑写进 Web 层。

| 文件 | 干什么 |
|------|--------|
| `rquant/strategy/base.py` | `Signal` 数据结构、`Strategy` 协议、MA/RSI/ATR/momentum/vol_ratio 等指标工具 |
| `rquant/strategy/registry.py` | `@register` 注册中心，按 name/category 管理策略 |
| `rquant/strategy/__init__.py` | 导入各策略触发注册，提供 `scan_stock()` / `scan_category()` / `scan_sell()`，导出全部指标函数 |
| `rquant/strategy/etf_rotation/` | ETF 轮动：`CrossBorderDca`（跨境定投）/ `DividendLowvolRotation`（红利低波轮动）/ `universe.py`（ETF 候选池） |
| `rquant/strategy/factor/` | 多因子：`MultiFactor`（V2 K 线 8 因子打分）/ `factor_calc.py`（V1 财务因子计算）/ `backtest_engine.py`（回测引擎） |
| `rquant/strategy/grid/` | 网格：`GridMartingale`（马丁格尔网格交易） |
| `rquant/strategy/legacy/` | 兼容版：`BuyHold`（买入持有）/ `ChanLun2B`（缠论二买） |
| `rquant/strategy/pattern/` | 形态识别：`DragonTigerPattern`（龙虎榜形态） |
| `rquant/strategy/router/` | 场景路由：`ScenarioRouter`（场景路由器）/ `market_regime.py`（市场状态判断） |
| `rquant/strategy/turtle/` | 海龟：`DonchianTurtle`（唐奇安通道突破） |
| `rquant/strategy/volume_breakout/` | 量价突破：`VpBreakout`（成交量价格突破） |

**Signal 字段：** `code, name, sector, strategy, category, current_price, suggested_buy, stop_loss, take_profit, reason, confidence, market_state, extra`
**导入即注册的策略（共 8 大类别，10 个策略）：**

| 类别 | 模块 | 策略类 |
|------|------|--------|
| `etf_rotation` | `cross_border_dca` | `CrossBorderDca` |
| `etf_rotation` | `dividend_lowvol_rotation` | `DividendLowvolRotation` |
| `factor` | `multi_factor` | `MultiFactor` |
| `grid` | `grid_martingale` | `GridMartingale` |
| `legacy` | `buyhold` | `BuyHold` |
| `legacy` | `chanlun2b` | `ChanLun2B` |
| `pattern` | `dragon_tiger` | `DragonTigerPattern` |
| `router` | `scenario_router` | `ScenarioRouter` |
| `turtle` | `donchian` | `DonchianTurtle` |
| `volume_breakout` | `vp_breakout` | `VpBreakout` |

`__init__.py` 导出指标函数：`atr, change_pct, highest, lowest, ma, momentum, prev_ma, rsi, vol_ratio`

### `rquant/business/` — 业务层

`__init__.py` 导出：`board, data, funds, market, portfolio, user`

| 文件 | 干什么 |
|------|--------|
| `data.py` | K 线 wrapper、内存股票快照、自选股/标的池兼容转发 |
| `pool_store.py` | SQLite 标的池和自选股持久化，替代老 `_DEFAULT_POOL` 文件内配置 |
| `portfolio.py` | 持仓、交易、快照 JSON 存储 |
| `funds.py` | 总资金、可用资金、已投入资金、已实现盈亏 |
| `board.py` | 东方财富真实行业/概念/地域板块排行与成分股 |
| `market.py` | 大盘指数数据，供场景路由器使用 |
| `system.py` | 市场状态、策略状态、内存系统日志 |
| `user.py` | 默认本地用户与模拟用户数据目录 |

数据池与标的池设计详见 `docs/数据池.md`。该文档记录当前 `pool` / `watchlist` / `DataSourcePool` / Parquet 的结构审查，以及面向多因子策略的主题池、参考池、基准池优化方案。

### `rquant/web/` 与前端

| 文件 | 干什么 | 改它的时候 |
|------|--------|-----------|
| `rquant/web/__init__.py` | 导出 `CATEGORY_LABELS` / `DEFAULT_PORT` / `create_app` / `run` | 对外接口变更 |
| `rquant/web/app_factory.py` | `create_app()` / `run()`，启动 MQ 和 Flask app | 启动方式或 Flask 配置变更 |
| `rquant/web/routes.py` | 首页、持仓、资金、板块、自选股、系统状态 API | 新增 API 或表单行为变更 |
| `rquant/web/views.py` | `_compute_treemap()` / `_build_watchlist_view()` / 表单安全转换 / 标的池名称映射 | 视图拼装或 Treemap 计算变更 |
| `templates/index.html` | 单页看板 HTML + JS | UI 交互变更 |
| `static/style.css` | 暗色主题样式 | 样式变更 |
| `app.py` | 根目录兼容启动入口 | 通常不改 |

**Web 层不拥有策略逻辑。** 路由做参数解析、调用 business/strategy、渲染模板或 JSON。

## 股票代码格式

小写 `sh`/`sz` + 6 位数字。例：`sh600519`、`sz000001`。

K 线 DataFrame 列名：`date, open, high, low, close, volume`（全小写）。Parquet 额外可有 `amount, turnover`。

## 陷阱

1. **根目录 `app.py` / `strategy.py` 是兼容入口**，新代码优先改 `rquant/`。
2. **Sina / 东方财富接口都会挂**，下游要容错空返回或 stale 缓存。
3. **板块不再是 ETF 代理**，当前 `board.py` 直接拉东方财富真实板块和成分股。
4. **自选股已迁移到 SQLite meta**，`data/watchlist.json` 只做老数据迁移。
5. **资金和持仓是两套存储**，买卖路由必须同时更新 `portfolio.py` 和 `funds.py`。
6. **前端 squarify 已删**，Treemap 坐标由 `rquant/web/views.py` 计算。
