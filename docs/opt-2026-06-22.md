**位置**：`rquant/web/routes.py:78-117`

**问题**：

### 2. 路由器全局缓存污染回测

**位置**：`rquant/strategy/router/market_regime.py:127-157`

**问题**：

```python
_REGIME_CACHE: dict[str, MarketState] = {}

def get_market_regime(index_df=None) -> MarketState:
    today = datetime.now().strftime("%Y-%m-%d")
    if index_df is not None:
        state = MarketRegime().detect(index_df)
        _REGIME_CACHE[today] = state   # ← 强制覆盖"今天"缓存
        return state
    if today in _REGIME_CACHE:
        return _REGIME_CACHE[today]    # ← 回测时拿不到历史状态
    ...
```

**影响**：

- `ScenarioRouter.signal_buy` 每次调用 `get_market_regime()` 不传 `index_df`，直接走全局缓存
- **回测场景**：用 `scripts/run_backtest.py` 跑 2020-01 → 2026-06，回测引擎**不会**调 `clear_regime_cache`，整次回测只用"今天"的 regime 判定
- 这是**真实的回测错误**——过去每个月的 BULL / BEAR / SIDEWAYS 状态被替换成"今天的"

**修复 patch**（market_regime.py）：

```python
def get_market_regime(
    index_df: pd.DataFrame | None = None,
    use_cache: bool = True,
) -> MarketState:
    """获取市场状态

    use_cache=False → 不读写缓存（回测场景必须用这个）
    use_cache=True  → 按日缓存（实盘场景性能优化）
    """
    today = datetime.now().strftime("%Y-%m-%d")

    if index_df is not None:
        # 强制重算 + 不污染"今天"缓存（用传入的日期作 key）
        cache_key = index_df["date"].iloc[-1] if not index_df.empty else today
        state = MarketRegime().detect(index_df)
        if use_cache:
            _REGIME_CACHE[cache_key] = state
        return state

    if use_cache and today in _REGIME_CACHE:
        return _REGIME_CACHE[today]

    # 缓存 miss —— 拉指数 K 线
    try:
        from rquant.business.data import fetch_kline
        df = fetch_kline("sh000001", 130)
    except Exception:
        df = None

    state = MarketRegime().detect(df)
    if use_cache:
        _REGIME_CACHE[today] = state
    return state
```

并修改 `ScenarioRouter.signal_buy`：

```python
def signal_buy(self, code, name, sector, df):
    ...
    # ★ 实盘路径用缓存；回测引擎会自己传 index_df 进来
    state = get_market_regime(use_cache=True)
    ...
```

`scenario_router` 增加回测友好的入口：

```python
def signal_buy_at(self, code, name, sector, df, regime_state: MarketState):
    """回测用：调用方传入已算好的 regime，避免污染全局缓存"""
    sub_cats = self._pick_subcategories(regime_state.regime)
    ...
```

并在 `scripts/backtest_multi_factor.py` 调每日时**预计算 regime**：

```python

index_df_until_dt = index_df[index_df["date"] <= dt]
regime_state = get_market_regime(index_df=index_df_until_dt, use_cache=False)
```

**收益**：回测不再被"今天的状态"污染；OOS 验证更可信。

### 3. 月频回测 look-ahead 隐患

**位置**：`rquant/strategy/factor/factor_calc.py:78-86`（`_extract_momentum`）+ `:88-120`（`_extract_factors`）

**问题**：

```python
def _extract_momentum(code: str, df: pd.DataFrame) -> Optional[float]:
    """20 日动量因子：现价 / 20日前收盘 - 1"""
    if df is None or len(df) < 21:
        return None
    try:
        return float(df["close"].iloc[-1] / df["close"].iloc[-21] - 1)
        # ★ df 已经是 250 天全量，不是"调仓日 ≤ rb_date"的切片
    except Exception:
        return None
```

`run_pipeline` 接收 `rebalance_date` 但只在硬过滤里用了（line 66-69 检查停牌），**没有**在 `_extract_momentum` 里切片。

**影响**：

- 月频回测用全量 K 线（默认 250 天）
- 当前调仓日 `rb_date` 是 `2025-06-01`，但 `df["close"].iloc[-1]` 是**最新一天的收盘价**（可能已经到 2026-06）
- 选股时用了"调仓日之后"的动量 → look-ahead bias

**修复 patch**（factor_calc.py）：

```python
def run_pipeline(
    snap_date: str,
    rebalance_date: str,
    kline_days: int = 250,
    top_n: int = DEFAULT_TOP_N,
) -> Optional[pd.DataFrame]:
    ...
    # 2) 拉取 K 线（全部股票）— 同时按调仓日切片
    kline_map: dict[str, pd.DataFrame] = {}
    for code in all_codes:
        df = fetch_kline(code, kline_days)
        if df.empty:
            continue
        # ★ 严格时序：只用 ≤ rebalance_date 的数据
        df = df[df["date"] <= rebalance_date].reset_index(drop=True)
        if df.empty:
            continue
        kline_map[code] = df
        fetched += 1
    ...
```

**收益**：月频回测不再有 look-ahead；与 `scripts/backtest_multi_factor.py` 的精细版保持一致。

## 🟡 P1 - 建议改（可维护性 / 性能）

### 4. 跨请求 K 线 LRU 复用

**位置**：新增 `rquant/data_source/kline_cache.py`

**问题**：

- `data_source/sina.py` 有 SQLite 缓存，但每次调用 `to_dataframe` 都要重新构造 DataFrame
- 首页 sell + buy 循环里，同一只股票可能（理论上）被拉两次
- 自选股刷新 / 路由 `/api/watchlist/analyze/<code>` 反复触发

**修复**：

```python
# rquant/data_source/kline_cache.py
from functools import lru_cache
import pandas as pd
from . import pool as ds_pool

@lru_cache(maxsize=256)
def fetch_kline_cached(code: str, days: int) -> tuple:
    """进程内 LRU 缓存 + 返回 tuple（pd.DataFrame 不可 hash）"""
    df = ds_pool.to_dataframe(code, days)
    # 把 df 序列化成 tuple of tuples（hashable）
    if df.empty:
        return ((), ())
    return tuple(map(tuple, df.values)), tuple(df.columns)

def fetch_kline_cached_df(code: str, days: int) -> pd.DataFrame:
    rows, cols = fetch_kline_cached(code, days)
    if not rows:
        return pd.DataFrame()
    return pd.DataFrame(rows, columns=list(cols))

def clear():
    fetch_kline_cached.cache_clear()
```

并在 `rquant/business/data.py:fetch_kline` 里加个开关：

```python
def fetch_kline(code: str, days: int = 250, use_cache: bool = True) -> pd.DataFrame:
    if use_cache:
        from rquant.data_source.kline_cache import fetch_kline_cached_df
        return fetch_kline_cached_df(code, days)
    return pool.to_dataframe(code, days)
```

**收益**：

- 单次首页 / 单次回测跑 8 因子 8 次时，DataFrame 构造只算一次
- 估算：单页节省 200-500ms（池子 20 只时）
