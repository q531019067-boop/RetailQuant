## TL;DR

把早期 4 因子 K 线实现扩展成 **8 因子 / 4 道硬过滤 / 横截面友好的完整版**：
- 动量 2 + 趋势 3 + 量价 3 = 8 因子
- 停牌 / 流动性 / ST / 上市天数 = 4 道硬过滤
- 新增 `score()` / `score_batch()` 方法，回测引擎能横截面排序选 TopN
- 1 年回测（2025-08 → 2026-06，12 只标的）：**+11.69% 总收益 / +13.93% 年化 / 61.5% 胜率 / -10.55% 最大回撤**
- 同窗口 5 只等权买入持有基准：**-4.75%**（超额 **+16.44%**）

关键发现：**每周调仓（freq=5）显著跑赢每日调仓**——弱市里频繁换手=摩擦成本+追涨杀跌。

## 1. 设计背景：为何扩展到 8 因子

| 维度 | 早期 4 因子 | 当前 8 因子 |
|------|----|----|
| 因子数 | 4（动量 / RSI / 量比 / 波动率） | **8**（动量 ×2 + 趋势 ×3 + 量价 ×3） |
| 过滤 | 仅 "数据不足" 软过滤 | **4 道硬过滤**（停牌 / 流动性 / ST / 上市天数） |
| 视角 | 时间序列打分 | **时间序列 + 横截面** |
| API | `signal_buy`（trigger 阈值后） | `score` / `score_batch` / `signal_buy` |
| 实战缺什么 | 趋势强度、量价共振、突破确认 | ✅ 全补 |
| 财面因子 | ⚠️ 占位 | ⚠️ 仍占位（需东财/聚宽） |

早期的核心问题：**只有"自己历史上是好的"判断，没有"相对强弱"判断**。多因子的精髓是横截面——在所有候选里 rank，选 TopN。当前实现把这条管道打通。

## 2. 8 因子详解

### 2.1 因子分组与权重

| 组 | 权重 | 因子 | 简称 | 算式 / 阈值 |
|----|------|------|------|------------|
| **动量** | 0.35 | M1 · 20 日动量 | momentum_20d | `(close / close_20d_ago − 1) × 100%` → clip(±10%) |
| | | M2 · 60 日动量 | momentum_60d | 同上，clip(±20%) |
| **趋势** | 0.35 | T1 · MA20 偏离 | ma20_bias | `(close / MA20 − 1) × 100%` → tanh(scale=5) |
| | | T2 · 多头排列 | ma_alignment | MA5 > MA10 > MA20 全满足 → +1；空头 → -1；混乱 → -0.3 |
| | | T3 · 60 日突破 | breakout_60d | 距 60 日高点的距离 → [0, 1] 线性 |
| **量价** | 0.30 | 5 日量比 | vol_ratio | `today_vol / 5d_avg_vol` → tanh(±1) |
| | | 量价共振 | vol_price_sync | 量比>1.2 + 涨幅>0 → 加分；反之减分 |
| | | V3 · 波动率惩罚 | volatility | 20 日 std/mean → 越高越扣分 |

**每个因子的输出都归一化到 [-1, +1]**，加权求和后 score 也在 [-1, +1]。

### 2.2 归一化函数

```python
_norm_clip(x, lo=-1, hi=1)          # 硬截断（动量类）
_norm_tanh(x, scale=1.0)            # 平滑归一化（MA 偏离、量比类）
#   → tanh(x / scale)：scale 越小越敏感
#   → x=0 → 0；x=±1 → ±0.76；x=±2 → ±0.96
```

两种归一化各有适用场景：
- **clip** 适合有明确边界的（如动量 ±10% 已经算极端）
- **tanh** 适合连续平滑的（如 MA 偏离理论上无界）

### 2.3 因子设计哲学

**为什么动量要分 20 + 60 两个？**
- 20 日捕捉短期主升浪启动
- 60 日确认中期趋势（避免追在 20 日反弹的鱼尾）
- 两个一起用 = "既要启动又要确认"

**为什么趋势组要 3 个因子？**
- T1 MA 偏离 = "现价相对 20 日均价的位置"
- T2 多头排列 = "短中期均线方向一致"
- T3 突破 = "距离前高多近"
- 三者互补：T1 看位置、T2 看方向、T3 看距离前高

**为什么量价要加共振因子？**
- 5 日量比只看资金面，没看方向
- 量价共振把"放量 + 上涨"和"缩量 + 下跌"双向都给分
- 这是 A 股最经典的"主力资金"信号

**为什么波动率惩罚（V3）是惩罚项不是加分项？**
- 高波动 = 风险高
- 弱市里 5%+ 波动率已经算妖股
- 扣分而不是给负权——保留"如果其它因子极强仍可触发"

### 2.4 因子权重为何这样分

```
动量 0.35 + 趋势 0.35 + 量价 0.30 = 1.00
```

- **动量 = 趋势 = 0.35**：动量是已发生的趋势，趋势是当前方向，两者同等重要
- **量价 0.30 略低**：因为量价是"聪明钱"的代理信号，容易被操纵（拉高出货 / 对倒）

注意：这是先验权重，**回测可以重新调**。当前权重经过因子扩展前后的对比实验后定下来的，不是网格搜索最优。

## 3. 4 道硬过滤

任一不过 → 该股直接 `score = -∞`，不参与排序：

| # | 过滤 | 规则 | 作用 |
|---|------|------|------|
| 1 | **停牌** | 当日 `volume == 0` | 防止"用旧价触发买入后无法成交" |
| 2 | **流动性** | 20 日均成交额 < 500 万 | 排除小盘仙股（滑点 / 操纵风险） |
| 3 | **ST / 退市** | 名称含 `ST` 或 `退` | 排除高风险股 |
| 4 | **上市天数** | K 线 < 60 天 | 排除新股 / 数据不足 |

**设计取舍**：

- **停牌 + 流动性是"可交易性"过滤**——没这两道会买到买不到的
- **ST + 上市天数是"风险性"过滤**——ST 股波动剧烈违反策略假设；新股 60 日均线都没成型
- 当前所有阈值都是类内 hardcode，未来可改 `MultiFactor` 类属性
- 没加"行业"过滤（中性化是改进方向）

## 4. 横截面 + 时间序列双视角

```
              ┌─────────────────────────────┐
              │  dt 当日所有候选             │
              │  (N 只)                      │
              └──────────────┬──────────────┘
                             │ 对每只算 8 因子（只看 ≤ dt 的 K 线）
                             ▼
              ┌─────────────────────────────┐
              │  4 过滤 → 过滤后剩 M 只     │
              └──────────────┬──────────────┘
                             │ 加权求和得 score
                             ▼
              ┌─────────────────────────────┐
              │  按 score 降序排             │
              │  取 Top K（= max_positions） │
              └─────────────────────────────┘
```

**关键**：每只标的的 score 只跟自己**过去**的 K 线比（时间序列），但**最终选股**是跟**当天所有候选**比（横截面）。这就是多因子 vs 单因子策略的本质区别。

## 5. API 设计

```python
from rquant.strategy.factor.multi_factor import MultiFactor

mf = MultiFactor()

# (1) 单只打分（带过滤）
score, detail = mf.score(df, name="士兰微", code="sh600460")
# score ∈ [-1, +1] ∪ {-∞}
# detail = {"filtered": bool, "factors": {...}, "weighted_components": {...}}

# (2) 批量打分 + 横截面 rank（回测引擎用）
results = mf.score_batch(
    {"sh600460": df1, "sh512480": df2, ...},  # code → DataFrame
    {"sh600460": "士兰微", "sh512480": "国泰CES半导体芯片ETF", ...}
)
# results = [{code, name, score, rank, rank_pct, factors, ...}, ...]
# 按 score 降序排，可直接取 top_N

# (3) 单只触发（保持旧 API 兼容）
sig = mf.signal_buy(code, name, sector, df)  # → Signal | None
sig.suggested_buy, sig.stop_loss, sig.take_profit
sig.reason  # 人话: "多因子综合 +0.67（8 因子）| Top3: M1 +0.40 / T1 +0.20 / T3 +0.18"
sig.extra   # 因子明细 dict
```

**为什么拆 3 个方法？**
- `score()`：回测引擎排 TopN 用
- `score_batch()`：横截面批量打分（一次算所有候选）
- `signal_buy()`：保持单只触发语义（routes.py 调它给前端展示）

## 6. 回测引擎

文件：`scripts/backtest_multi_factor.py`

### 6.1 严格时序保证

```python
# ✅ 正确：dt 决策只用 ≤ dt 的数据
df_at_dt = hist[hist["date"] <= dt]
sigs = mf.score_batch({c: df[df["date"] <= dt] for c, df in all_klines.items()})

# ❌ 错误：dt 决策用到未来数据（look-ahead bias）
sigs = mf.score_batch(all_klines)  # all_klines 含 dt 之后的数据
```

回测引擎在 `run()` 里**每一天**都对每只标的做 `df[df["date"] <= dt]` 切片。

### 6.2 资金管理

- 起始资金：100 万（可配 `--capital`）
- 最大持仓：5 只（可配 `--positions`）
- 单只仓位：`cash × 97% / max_positions`（留 3% 现金缓冲）
- 买入按 A 股 100 股一手向下取整
- 现金不够时自动减仓到能买得起的最小手数

### 6.3 交易费用

```python
commission_rate = 0.00025   # 佣金万 2.5（最低 5 元）
stamp_rate = 0.001          # 印花千 1（仅卖出）

def _fee(side, amount):
    commission = max(amount * 0.00025, 5.0)
    stamp = amount * 0.001 if side == "SELL" else 0
    return commission + stamp
```

- 单边总费用约 0.125% / 0.225%（买/卖）
- 100 万一年换手 50 次 ≈ 0.6% 总费用（不算印花再翻倍）

### 6.4 风控三层

| 层 | 触发 | 动作 |
|----|------|------|
| 止盈 | `pnl_pct >= +15%` | 卖出 |
| 止损 | `pnl_pct <= -8%` | 卖出 |
| 满期 | `持仓 ≥ 21 个交易日` | 卖出 |
| 排名跌出 | 不在 TopN | 卖出 |

**前 3 个是单只的风控**（基于成本），**第 4 个是横截面的风控**（基于排名）。

### 6.5 T+1 模拟

- 当日收盘价决策 → 次日开盘价成交（**当前用收盘价近似**）
- 这是 A 股最严的约束——T+1 不能当日买卖
- 改进方向：分钟级数据 + 用次日 open 价成交

### 6.6 输出文件

```bash
uv run python scripts/backtest_multi_factor.py \
    --capital 1000000 --positions 5 --start 2025-08-01
```

生成 3 个文件到 `results/`：

| 文件 | 内容 |
|------|------|
| `backtest_multi_factor_trades.csv` | 每笔买卖（日期/方向/价格/盈亏/原因） |
| `backtest_multi_factor_equity.csv` | 每日净值 / 现金 / 持仓市值 |
| `backtest_multi_factor_summary.json` | 业绩指标汇总 |

## 7. 回测结果（核心）

### 7.1 默认配置：1 年 / 12 只标的 / 每周调仓

| 指标 | MultiFactor | 基准（5 只买入持有） |
|------|-----------|----------------------|
| 窗口 | 2025-08-01 → 2026-06-18（212 个交易日） | 同上 |
| 起始资金 | ¥1,000,000 | ¥1,000,000 |
| **终值** | **¥1,116,910** | ¥952,500 |
| **总收益** | **+11.69%** | -4.75% |
| **年化** | **+13.93%** | — |
| **胜率** | **61.5%**（32 盈 / 20 亏） | — |
| **最大回撤** | -10.55% | — |
| 交易笔数 | 52 笔 | — |
| 平均持仓 | 14.7 天 | — |

**超额收益 +16.44%**（在弱市大盘下做到这一点很关键）。

### 7.2 调仓频率扫描（关键实验）

固定参数：max_pos=5, BUY=0.50, MIN=60。**只改调仓频率**：

| freq | 总收益 | 年化 | 胜率 | 笔数 | 平均持仓 | 回撤 |
|------|--------|------|------|------|----------|------|
| 1（每日） | -0.75% | -0.89% | 39.0% | 82 | 10.7 天 | -15.13% |
| 2 | +0.03% | +0.04% | 41.9% | 62 | 13.3 天 | -13.29% |
| 3 | -2.38% | -2.81% | 41.4% | 58 | 13.3 天 | -10.68% |
| **5（每周）** ⭐ | **+11.69%** | **+13.93%** | **61.5%** | 52 | 14.7 天 | **-10.55%** |
| 7 | -1.89% | -2.23% | 52.4% | 42 | 14.4 天 | -12.74% |
| 10 | -3.32% | -3.90% | 38.9% | 36 | 17.4 天 | -15.43% |
| 15（半月） | +7.34% | +8.71% | **64.3%** | 28 | 24.8 天 | **-6.28%** |
| 20 | -13.35% | -15.54% | **20.0%** | 20 | 24.2 天 | **-20.67%** |

**核心结论**：
- **freq=5（每周）= 最佳**——吃到主升浪 + 控制摩擦成本
- **freq=1（每日）= 明显差**——频繁换手吃掉利润
- **freq=15（半月）= 胜率最高 64.3% + 回撤最小 -6.28%**——但反应太慢错过一波
- **freq=20 = 崩**——调仓太慢信号失效

### 7.3 持仓数扫描

固定 freq=5, BUY=0.50：

| max_pos | 总收益 | 胜率 | 回撤 |
|---------|--------|------|------|
| 3 | -1.09% | 50.0% | **-29.49%** |
| **5** ⭐ | **+11.69%** | **61.5%** | -10.55% |
| 7 | +3.56% | 56.0% | -12.27% |
| 10 | +2.28% | 56.0% | -8.97% |

**max_pos=3 太集中**（单股波动拖累）；5 / 7 / 10 都还行但 5 最优。

### 7.4 买入阈值扫描

| BUY | 总收益 | 胜率 | 笔数 |
|-----|--------|------|------|
| 0.60 | +0.02% | 38.7% | 62 |
| **0.50** ⭐ | **+11.69%** | **61.5%** | 52 |
| 0.40 | -10.09% | 35.3% | 102 |

BUY=0.5 是 sweet spot：太低触发多=噪声多；太高触发少=漏信号。

## 8. 改进方向

### 8.1 数据层

- [ ] 拉 **2-3 年** K 线（当前只 1 年，2025-08 起）

## 9. 局限与风险

### 9.1 当前回测的局限

| 局限 | 影响 | 优先级 |
|------|------|--------|
| 数据窗口仅 1 年 | 单年偏差风险 | P0 |
| 12 只标的偏集中 | 行业分散不足 | P1 |
| 财务因子未接入 | 缺少价值维度 | P1 |
| 6/15-18 士兰微单只贡献 ~3.5% | **胜率/收益严重依赖妖股** | P0 |
| 52 笔交易样本小 | 胜率 61.5% 统计意义有限 | P0 |
| 无分红再投资 | 长期收益低估 | P2 |

### 9.2 实战风险

- **未来数据使用风险**：所有因子只用 ≤ dt 的数据，但**调参过程**可能引入了隐式 look-ahead
- **过拟合风险**：8 因子 + 4 过滤 + 5 参数 = 17 个可调维度，1 年 52 笔样本极易过拟合
- **生存者偏差**：当前池子里的标的（如士兰微）是"曾经涨过的"——可能不代表未来

### 9.3 任何回测都适用的话

> 过去的表现不代表未来的收益。
> 这个回测告诉你的是"**在 A 股弱市下，多因子选股 + 每周调仓**比买入持有强"，**不是**"你实盘也能拿到 +11.69%"。

### 11.2 调参实验

```python
# 频率扫描
import sys
sys.path.insert(0, '.')
from scripts.backtest_multi_factor import BacktestEngine
from rquant.business import data
from rquant.strategy.factor.multi_factor import MultiFactor

pool = data.get_pool()
for freq in [1, 3, 5, 10, 15, 20]:
    mf = MultiFactor()
    eng = BacktestEngine(strategy=mf, max_positions=5)
    r = eng.run(pool=pool, start_date='2025-08-01', end_date='2026-06-18',
                rebalance_freq=freq, verbose=False)
    print(f'freq={freq:>2}  total={r.total_return_pct:+6.2f}%  '
          f'win={r.win_rate_pct:5.1f}%  trades={r.total_trades:>3}  '
          f'max_dd={r.max_drawdown_pct:6.2f}%')
```
